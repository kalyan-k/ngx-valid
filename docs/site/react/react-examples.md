# React Examples

The React demo uses the workspace package and shared Demo Platform shell.

## State-management comparison

[Open State Management Demos](http://127.0.0.1:4204/state/local-state)

Local State, Redux Toolkit, Zustand, Jotai, Recoil, MobX, and Context API each expose Home, Simple, Complex, and Performance pages backed by one shared validation contract. Use the left navigation to compare identical behavior without duplicating policies or form UI.

## Simple form

[Open Simple Form](http://127.0.0.1:4204/state/local-state/simple)

Demonstrates local controlled state, required/email/phone/date/checkbox rules, blur and change validation, submit, inline messages, summary links, focus management, valid/invalid status, and reset.

Related guides: [Quick Start](/docs/react-quick-start), [Field Validation](/docs/react-field-validation), and [Form Validation](/docs/react-form-validation).

## Complex form

[Open Complex Form](http://127.0.0.1:4204/state/local-state/complex)

Demonstrates nested personal data, multiple addresses and contact methods, conditional secondary email, generated policies, three groups, dynamic add/remove actions, programmatic validation, sample data, save, and reset.

Related guides: [Validation Policies](/docs/react-policies), [Validation Groups](/docs/react-groups), and [Dynamic Fields](/docs/react-dynamic-fields).

## Performance form

[Open Performance Form](http://127.0.0.1:4204/state/local-state/performance)

Demonstrates 80 generated fields, four groups, a generated large policy, validate-all, populate-valid, populate-invalid, reset, live timing, run count, page render count, and current errors.

Related guide: [Performance](/docs/react-performance).
