Performance Form — Dynamic Validation Benchmark

How this demo works
  • Configure the number of sections (1–50) and controls per section (1–200), then click Generate form.
  • Random controls — each slot gets a random HTML control type (text, email, number, date, checkbox, select, textarea, radio).
  • Grouped validation — each section is its own policy and form group, aggregated by the performance policy group (same pattern as the complex form).
  • Benchmarks — generation, render, validate-all, and submit timings are captured below.
  • Seed — use the same seed to reproduce the same random control layout across runs.
