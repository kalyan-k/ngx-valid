Complex Form — Multiple Policies on One Page

How this demo works
  • Three policies — PersonalInfo, ShippingAddress, and BillingAddress run independently on the same model.
  • Conditional required (*) — e.g. City/State/ZIP show * only after Street Address is entered; billing fields show * only when "Same as shipping" is unchecked.
  • Section badges — each card shows Not validated until Submit; hover for an explanation.
  • Page-level status — policy-validation-policy-group-status aggregates all sections via the registered checkout policy group.
  • Errors — field errors on blur; section and page summaries appear after Submit.
